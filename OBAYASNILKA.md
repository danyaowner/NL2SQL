# Часть 1: ГЛОССАРИЙ ТЕРМИНОВ (СЛОВАРЬ)

Здесь собраны все сложные слова из курсовой работы — от А до Я.
Каждый термин объяснён простым языком, с примерами.

---

## А

### API (Application Programming Interface)

**Простыми словами: Это как официант в ресторане.**
Вы говорите официанту, что хотите заказать, он передаёт это повару, а потом приносит вам готовое блюдо.
API — это такой 'официант' между программами. Одна программа отправляет запрос через API, другая обрабатывает его и возвращает ответ.

В курсовой: Мы используем OpenAI API — отправляем запрос серверу OpenAI, а в ответ получаем готовый SQL-запрос.

---
## Б

### Бенчмарк (Benchmark)

**Простыми словами: Контрольная работа для нейросетей.**
Это стандартный набор заданий, по которому проверяют, насколько хорошо работает система.
Все системы проходят один и тот же тест, чтобы можно было сравнить их качество.

В курсовой: Spider, WikiSQL, Bird — это бенчмарки для NL2SQL.
Они содержат тысячи пар 'вопрос на русском/английском → правильный SQL-запрос'.
Если система выдаёт правильные SQL 70 раз из 100 — значит, точность 70%.

### BERT

**Простыми словами: Нейросеть-пониматель.**
BERT читает текст и понимает смысл каждого слова в контексте.
Например, в предложении 'ключ от машины' и 'ключ к решению' слово 'ключ' означает разное. BERT это понимает.

В курсовой: BERT используется для понимания схемы базы данных — какие таблицы и колонки важны для запроса.

---
## Г

### Генерация SQL

**Простыми словами: Автоматическое создание SQL-запроса.**
Вы говорите 'найди отдел продаж', а программа сама пишет SQL-запрос, который это сделает.

В курсовой: Вместо ручного написания 'SELECT * FROM employees WHERE department = "Продажи"' вы просто пишете 'покажи продавцов'.

### GROUP BY

**Простыми словами: Сгруппировать по общему признаку.** Как разложить конфеты по цветам.
Пример: 'Средняя зарплата по отделам' → GROUP BY department сгруппирует сотрудников по отделам и для каждого посчитает среднюю.

---
## Д

### Датасет (Dataset)

**Простыми словами: Набор данных.**
Коллекция примеров, на которых нейросеть учится или проверяется. Как сборник задач по математике.

В курсовой: Spider (10 181 пример) — сборник вопросов с правильными SQL-запросами.

---
## Е

### Естественный язык (Natural Language, NL)

**Простыми словами: Обычный человеческий язык.**
Русский, английский — то, на чём мы говорим. В отличие от 'искусственных' языков вроде Python или SQL.

---
## И

### Интенция (Intention)

**Простыми словами: Что пользователь на самом деле хочет сделать.**
Когда вы говорите 'тут жарко' — ваша интенция не описать погоду, а попросить включить кондиционер.

В курсовой: Понять интенцию = понять, хочет пользователь найти данные (SELECT), посчитать (COUNT) или вычислить среднее (AVG).

---
## К

### Колонка (Column)

**Простыми словами: Столбец в таблице.**
В Excel это столбец A, B, C — в каждом данные одного типа: имена, зарплаты, даты.

---
## Л

### LLM (Large Language Model — Большая языковая модель)

**Простыми словами: Умный автодополнитель текста.**
Вы замечали, как телефон предлагает следующие слова при печати? Большая языковая модель — супер-мощная версия этого.
Она обучена на миллионах книг, статей и сайтов и умеет не просто дописывать слова, а вести осмысленный диалог.
Примеры: ChatGPT (GPT-4), Claude, YandexGPT.

В курсовой: LLM получает запрос 'найди всех сотрудников отдела разработки' и генерирует SQL.
Модель делает это не потому что 'понимает' SQL, а потому что видела миллионы SQL-запросов в обучении.

### LSTM (Long Short-Term Memory — Долгая краткосрочная память)

**Простыми словами: Нейросеть с памятью.** Обычная нейросеть 'забывает' начало предложения. LSTM помнит важное надолго.

---
## М

### Метрика

**Простыми словами: Измеритель качества.** Как оценка в школе: 5 = отлично, 2 = плохо.

В курсовой:
- **EX (Exact Match)** — точное совпадение SQL слово-в-слово?
- **EA (Execution Accuracy)** — если выполнить SQL, результат правильный? (Даже если SQL написан иначе, но результат верен)

### Модульная архитектура

**Простыми словами: Разделение программы на отдельные блоки (как LEGO).**
Каждый блок делает своё. Можно менять один блок, не трогая другие.

---
## Н

### Нейронная сеть (Neural Network)

**Простыми словами: Математическая модель, которая учится на примерах.**
Как ребёнок, который учится отличать кошек от собак по 1000 фотографий.

### NL2SQL (Natural Language to SQL)

**Простыми словами: Переводчик с человеческого языка на язык баз данных.**
Вы пишете 'покажи всех с зарплатой больше 100 тысяч' → программа выдаёт SQL-запрос.

---
## П

### Промпт (Prompt)

**Простыми словами: Инструкция для нейросети.**
Когда вы пишете ChatGPT 'напиши письмо начальнику' — это и есть промпт.

В курсовой: Промпт включает: (1) описание схемы БД, (2) примеры 'вопрос→SQL', (3) сам запрос.
Чем лучше промпт, тем точнее SQL.

### Промпт-инжиниринг (Prompt Engineering)
**Искусство правильно задавать вопросы нейросети.** Как правильно формулировать запрос в Google.

---
## Р

### RAG (Retrieval-Augmented Generation)
**Простыми словами: Поиск + генерация.**
Сначала ищем похожие примеры, потом отправляем запрос нейросети вместе с ними.
Как шпаргалка на экзамене.

### Реляционная база данных
**Данные в связанных таблицах.** Таблицы ссылаются друг на друга.
Наша БД: employees, projects, tasks, comments — все связаны.

---
## С

### Self-Attention (Само-внимание)
**Механизм 'внимания' нейросети.** Понимает, какие слова связаны между собой.
Например: 'Она взяла книгу, которую купила вчера' — 'которую' относится к 'книга'.

### SQL (Structured Query Language)
**Язык общения с базами данных.** Команды: SELECT, FROM, WHERE.

### Streamlit
**Конструктор веб-страниц для Python.** Позволяет сделать веб-интерфейс за 10 минут без HTML/CSS.

---
## Т

### Таблица
**Как лист Excel.** Строки и столбцы для хранения данных.
У нас 4 таблицы: employees, projects, tasks, comments.

### Transformer
**Архитектура нейросети, которая произвела революцию.**
Изобретён Google в 2017. Основа для GPT, ChatGPT, BERT.

---
## Ф

### Fine-tuning (Донастройка)
**Дообучение готовой нейросети.** Как повар, который знает всё, но доучивается на итальянской кухне.

### Few-shot Learning
**Показываем 2-3 примера, и нейросеть понимает задачу.** Как показать ребёнку 3 примера умножения.

---
## Э

### Эмбеддинги (Embeddings)
**Числовое представление смысла слов.** Превращаем 'собака' в набор чисел, отражающий смысл.
Похожие слова → похожие числа.

---
## Дж

### JOIN
**Склеивание таблиц.** Если у вас две таблицы — 'сотрудники' и 'задачи' — JOIN соединяет их.

# Часть 2: КАК РАБОТАЕТ ПРОТОТИП (ПРОСТЫМИ СЛОВАМИ)

## 2.1. Что делает программа? (Общая картина)

Представьте, вы пришли в библиотеку. Вы говорите библиотекарю: «Дайте мне все книги по физике, которые взяли в прошлом месяце». Библиотекарь идёт, проверяет записи и приносит список.

**Наша программа — это такой библиотекарь, но для базы данных.**

Вы пишете по-русски или по-английски: «Найди всех сотрудников отдела разработки».
Программа сама превращает это в SQL-запрос, выполняет его и показывает результат.

Вам НЕ нужно знать SQL. Вы просто задаёте вопрос на обычном языке.

---

## 2.2. Пошагово: что происходит внутри программы

Допустим, вы ввели: **«Сколько задач в проекте CRM Platform?»**

### Шаг 1. Анализ (nl_module.py — модуль естественного языка)

Программа делает с вашим текстом три вещи:

**① Очищает запрос** — убирает лишние пробелы, знаки препинания (но оставляет смысл)
**② Определяет язык** — русский или английский (чтобы искать ключевые слова правильно)
**③ Понимает, что вы хотите** — смотрит на ключевые слова:
   - «Сколько», «количество» → значит, нужен COUNT (подсчёт)
   - «Найди», «покажи» → SELECT (выборка)
   - «Средняя», «среднее» → AVG (среднее значение)

➡ **Результат:** программа поняла, что вы хотите ПОСЧИТАТЬ задачи.

### Шаг 2. Выбор таблиц (schema_selector.py — селектор схемы)

Программа смотрит: о каких таблицах идёт речь?

У нас в базе 4 таблицы:
| Таблица | Содержит |
|---------|----------|
| employees | Сотрудники (имя, отдел, зарплата) |
| projects | Проекты (название, бюджет, статус) |
| tasks | Задачи (название, проект, статус) |
| comments | Комментарии к задачам |

В запросе есть слова «задачи» и «проект» → программа выбирает таблицы tasks и projects.

➡ **Результат:** программа знает, с какими таблицами работать.

### Шаг 3. Генерация SQL (sql_generator.py — генератор SQL)

Тут есть два режима:

**Режим DEMO (простой):** Программа использует правила-шаблоны.
Она проверяет: «Сколько» → COUNT. «В каждом проекте» → GROUP BY.
И собирает SQL как конструктор.
Например, для запроса «Сколько задач в каждом проекте» получится:
```sql
SELECT p.project_name, COUNT(t.task_id) AS task_count
FROM projects p
LEFT JOIN tasks t ON p.project_id = t.project_id
GROUP BY p.project_name;
```
⚠️ **Важно:** DEMO-режим может НЕ понять конкретное название проекта (например, «CRM Platform»)
и вернёт задачи по ВСЕМ проектам сразу. Это нормально — для точных названий нужен FULL-режим.

**Режим FULL (умный):** Программа отправляет запрос в ChatGPT (через OpenAI API).
ChatGPT получает: описание БД + примеры «вопрос→SQL» + ваш запрос.
ChatGPT сам думает и возвращает правильный SQL.

➡ **Результат:** готовый SQL-запрос.

### Шаг 4. Проверка (validator.py — валидатор)

Прежде чем выполнять, программа проверяет:
- **Синтаксис:** SQL написан правильно? Нет ли опечаток?
- **Безопасность:** Запрос не удаляет данные? (Защита от DROP, DELETE, UPDATE)
- **Таблицы:** Указанные таблицы существуют в базе?

Если всё плохо → программа говорит «Ошибка» и не выполняет.
Если всё хорошо → идём дальше.

➡ **Результат:** SQL безопасен и корректен.

### Шаг 5. Выполнение (init_db.py + sqlite3)

Программа подключается к файлу test_company.db (наша база данных).
Выполняет SQL-запрос. Получает результат.

➡ **Результат:** таблица с данными — например:
| project_name | task_count |
|-------------|-----------|
| CRM Platform | 5 |
| Mobile App | 4 |
| Data Warehouse | 3 |

### Шаг 6. Показ (app.py — интерфейс на Streamlit)

Вся магия отображается на веб-странице в вашем браузере:
- Ваш вопрос
- Сгенерированный SQL-код (можно скопировать)
- Результат в виде таблицы

## 2.4. Разница между DEMO и FULL режимами

| Характеристика | DEMO (простой) | FULL (через API) |
|---------------|----------------|------------------|
| Нужен интернет? | ❌ Нет | ✅ Да |
| Нужен API-ключ? | ❌ Нет | ✅ Да (от OpenAI) |
| Стоит денег? | ❌ Бесплатно | ✅ ~1 цент за запрос |
| Понимает сложные запросы? | ❌ Только простые | ✅ Любые |
| Точность | ~60-70% | ~90%+ |
| Скорость | Мгновенно | 2-5 секунд |

**DEMO-режим** работает как таблица умножения: если запрос подходит под шаблон — ответ верный. Если нет — ошибка.
**FULL-режим** работает как репетитор: понимает любую формулировку.

## 2.5. Как читать код прототипа (если захотите)

Откройте папку **prototype/** в редакторе. Файлы расположены от простого к сложному:

**1️⃣ validator.py** (∼30 строк) — самый простой файл
Просто проверяет SQL на ошибки. Идеально для первого чтения.

**2️⃣ init_db.py** (∼60 строк) — создание базы данных
Показывает, как выглядят таблицы и как вставлять данные.

**3️⃣ nl_module.py** (∼150 строк) — главный «мозг»
Самое интересное. Тут происходит вся магия анализа текста.

**4️⃣ schema_selector.py** (∼80 строк) — выбор таблиц
Простой, но важный модуль.

**5️⃣ sql_generator.py** (∼200 строк) — генерация SQL
Самый сложный файл. Не пугайтесь — большая часть это шаблоны.

**6️⃣ app.py** (∼100 строк) — интерфейс
Склеивает всё вместе и показывает результат.

## 2.6. Что делать, если программа выдала не то?

1. **Переформулируйте запрос** — вместо «сотрудники отдела разработки» попробуйте «покажи всех, кто работает в разработке»
2. **Проверьте язык** — если запрос на русском, программа ищет русские ключевые слова. Смешивать языки не стоит
3. **Используйте английский** — для DEMO-режима английские запросы работают точнее
4. **Включите FULL-режим** — если у вас есть API-ключ OpenAI, программа станет значительно умнее

# Часть 3: ЧТО ТАКОЕ API И ЗАЧЕМ ОН НУЖЕН

## 3.1. Аналогия: пульт от телевизора

Вы нажимаете кнопку «+» на пульте — громкость увеличивается.
Вы понятия не имеете, как внутри телевизора работает схема управления громкостью.
Вам это и не нужно — пульт делает всю сложную работу за вас.

**Пульт — это и есть API.**

Телевизор = сервер OpenAI с нейросетью.
Вы нажали кнопку = наша программа отправила запрос.
Громкость изменилась = мы получили готовый SQL.

В Части 1 была другая аналогия (официант). Выберите ту, которая понятнее лично вам.

## 3.2. Что такое API технически?

API = Application Programming Interface (программный интерфейс).
Это способ для двух программ общаться друг с другом.

Пример: приложение погоды спрашивает через API сервер, какая температура.
Сервер отвечает. Приложение показывает ответ на экране.
Вы не знаете, как сервер получает данные - вам это и не нужно.

## 3.3. OpenAI API в нашей курсовой

OpenAI - компания, создавшая ChatGPT.
Через их API мы отправляем запрос и получаем готовый SQL.

Схема: Запрос -> Наш прототип -> OpenAI API -> ChatGPT -> SQL -> Нам

Что мы отправляем: описание БД + примеры вопрос-правильный SQL + запрос
Что получаем: готовый SQL-запрос

## 3.4. Зачем нужен API?

Сделать нейросеть самому: годы учёбы, миллионы примеров, суперкомпьютер.
Использовать API: 5 минут на регистрацию, 1 строка кода, 2 секунды ответ.

API позволяет использовать сложные технологии без их создания.

## 3.5. API в повседневной жизни

- Карты на телефоне = Google Maps API
- Вход через Google = OAuth API
- Оплата картой = API платёжной системы
- Боты в Telegram = Telegram Bot API
- YouTube на сайте = YouTube API

## 3.6. API-ключ OpenAI

1. Зайти на platform.openai.com
2. Создать API Key (sk-...xyz)
3. Хранить как пароль от банка - никому не показывать!
4. Один запрос GPT-4o стоит примерно 1 цент
5. Для тестов хватит 5 долларов

Использовать: set OPENAI_API_KEY=sk-...ваш_ключ...
